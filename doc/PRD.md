# Bakubaku 产品需求文档（PRD）

> Bakubaku — 取名自海贼王吞吞果实（Baku Baku no Mi），吃掉磁盘上的垃圾。

## 1. 产品概述

### 1.1 产品定位

Bakubaku 是一款 AI 驱动的智能磁盘清理工具，定位为**磁盘文件管家**。

与传统清理工具的核心区别：现有工具解决"看见"问题（哪些文件占了空间），Bakubaku 解决"判断"问题（这个文件到底能不能删）。通过大模型理解文件语义，用人话告诉用户每个文件是什么、删了有没有影响。

### 1.2 目标用户

| 用户群 | 使用方式 | 核心诉求 |
|--------|---------|---------|
| 普通用户 | GUI 图形界面 | 不懂技术也能安心清理，不怕删错 |
| 极客/开发者 | CLI 命令行 | 快速高效，可脚本化，灵活控制 |

### 1.3 产品原则

- **用户启动才工作** — 不后台常驻、不自启动、不弹窗，绝不做 360 式流氓软件
- **隐私优先** — 所有分析本地完成，不上传文件内容和路径到云端
- **安全第一** — 宁可漏删不可误删，清理可撤销
- **越用越智能** — 记忆系统学习用户磁盘特征和偏好，后续扫描越来越快越准

## 2. 核心功能

### 2.1 秒级全盘扫描

**描述：** 启动后快速扫描全盘，建立文件树快照。

**功能要求：**
- 利用 NTFS MFT 直读技术实现秒级扫描速度
- 首次扫描建立完整快照，后续启动基于记忆系统做增量扫描
- 扫描过程有实时进度反馈（当前目录、已扫描文件数、预估剩余时间）
- 权限不足的目录自动跳过并记录，不中断扫描
- 符号链接默认不跟随，避免循环引用

**性能目标：**
- NVMe SSD：100 万文件 < 5 秒
- SATA SSD：100 万文件 < 15 秒
- 机械硬盘：100 万文件 < 60 秒

### 2.2 三层智能分类引擎

**描述：** 采用漏斗式分层策略，对文件进行分类和重要性评级。

**第一层：规则引擎（覆盖 ~90% 文件）**
- 基于预定义规则快速分类已知垃圾和已知重要文件
- 规则类别：系统临时文件、构建缓存、浏览器缓存、包管理器缓存、日志文件等
- 白名单保护：系统文件、用户文档等绝不触碰
- 规则以配置文件形式管理，可扩展

**第二层：启发式分析（覆盖 ~5% 文件）**
- 文件熵值检测
- 目录结构语义分析（如父目录名含 "backup_old_2019"）
- 文件访问时间分析
- 文件头 magic bytes 检测

**第三层：LLM 智能分析（覆盖 ~5% 文件）**
- 仅处理前两层无法判定的文件
- 只发送文件元数据（路径、大小、时间戳），绝不发送文件内容
- 路径脱敏处理（用户名等敏感信息替换）
- 批量打包请求（200-500 个文件一批），控制成本
- 分析结果缓存到本地，相同模式复用结论

**输出分级：**

| 等级 | 颜色 | 含义 | 默认操作 |
|------|------|------|---------|
| Level 1 | 绿色 | 安全删除（临时文件、缓存） | 自动勾选 |
| Level 2 | 黄色 | 建议删除（旧日志、过期备份） | 默认勾选，可取消 |
| Level 3 | 橙色 | 可能不需要（LLM 判定） | 默认不勾选，展示理由 |
| Level 4 | 红色 | 请谨慎（可能重要） | 不可勾选，仅展示 |
| Level 5 | 锁定 | 受保护（系统文件、用户文档） | 不可见/锁定 |

### 2.3 AI 人话解释

**描述：** 每个文件/文件夹旁附带一句通俗易懂的解释。

**示例：**
- "这是微信的图片缓存，删除后微信会重新下载需要查看的图片，不影响聊天记录"
- "这是 Word 崩溃时留下的临时文件，删掉完全没问题"
- "这是你 2023 年安装 Unity 后留下的缓存，已经 2 年没用过了"
- "这是你的个税申报表，建议保留"

**要求：**
- 语言风格：口语化、无技术术语
- 长度：一句话，不超过 50 字
- 必须说明删除后的影响

### 2.4 卡片式渐进清理

**描述：** 扫描结果不以文件列表展示，而是按场景分成可消化的卡片。

**卡片类型示例：**
- "应用缓存 — 3.2 GB"（浏览器缓存、微信缓存等）
- "临时文件 — 1.5 GB"（系统临时文件、崩溃残留）
- "重复文件 — 1.8 GB"（相似照片、多版本文档）
- "已卸载软件残留 — 2.1 GB"
- "超过一年没打开的大文件 — 5 GB"
- "构建缓存 — 8 GB"（node_modules、target、__pycache__）

**交互流程：**
1. 扫描完成后展示所有卡片，每张标注可释放空间
2. 用户点击某张卡片 → 展开查看详细文件列表 + AI 解释
3. 用户勾选/取消勾选 → 确认清理
4. 清理完成 → 展示释放了多少空间 + 成就反馈

### 2.5 智能重复文件检测

**描述：** 不仅检测完全相同的文件，还能识别"几乎相同"的文件。

**检测能力：**
- 精确重复：基于文件 hash 的完全匹配
- 相似照片：连拍照片、不同分辨率的同一图片、截图
- 多版本文档：报告v1.docx、报告v2.docx、报告_final.docx、报告_final_final.docx
- 相同内容不同格式：如同一张图的 PNG 和 JPG 版本

**展示方式：**
- 缩略图网格预览
- 每组重复文件标注推荐保留项（最新/最高质量）
- 用户选择保留哪个，其余清理

### 2.6 已卸载软件残留清理

**描述：** 检测已卸载软件遗留在磁盘上的文件。

**检测范围：**
- AppData 中的配置文件夹
- ProgramData 残留
- 注册表残留项
- 开始菜单快捷方式残留
- 桌面快捷方式残留

**AI 加持：**
- 关联解释残留属于哪个已卸载软件
- 判断残留是否包含用户数据（如游戏存档），提示用户决策

### 2.7 清理可撤销（隔离区）

**描述：** 所有清理操作先移到隔离区，而非直接永久删除。

**机制：**
- 清理 = 移动到 Bakubaku 隔离目录
- 默认保留 7 天，到期后用户下次启动时提示是否永久删除
- 用户可随时从隔离区恢复文件
- 隔离区本身可设置空间上限，超出时优先清理最早的隔离文件

### 2.8 记忆系统

**描述：** 首次使用学习磁盘布局，后续越用越快越准。

**记忆内容：**
- 文件树快照（路径、大小、时间戳）— 用于增量扫描
- 已识别的软件及其缓存位置
- LLM 分析结果缓存（相同路径模式复用结论）
- 用户清理偏好（每次都跳过的目录、总是清理的类型）
- 用户确认/否决的判断（"用户说这个目录不能删" → 永久学到）

**学习效果：**
- 首次扫描：全量遍历 + LLM 大量分析，耗时较长
- 第二次：增量扫描 + 大量缓存命中，速度提升 80%+
- 长期使用：几乎不需要 LLM 调用，秒级完成分析

### 2.9 操作日志与审查

**描述：** 完整记录每次操作，供用户审查。

**日志内容：**
- 每次扫描的时间、范围、发现的文件数量
- 每个清理操作的详细记录（文件路径、大小、分类理由、AI 解释）
- 用户确认/否决的决策记录
- 从隔离区恢复的记录

**展示方式：**
- GUI 中提供"历史记录"页面，按时间线展示
- CLI 中支持 `bakubaku log` 命令查看
- 支持导出为文件（CSV/JSON）

## 3. 界面形态

### 3.1 GUI（图形界面）

面向普通用户，开箱即用、零学习成本。

**设计原则：**
- 类似手机清理大师的直觉化交互
- 文件分类用人话描述（"应用缓存"而非"node_modules"）
- 愉悦感设计：扫描动效、清理前后对比、成就反馈
- 技术栈：Tauri 2 + React + TypeScript

**主要页面：**
- 首页/仪表盘：磁盘空间概览 + 一键扫描
- 扫描结果：卡片式清理建议
- 文件详情：单个文件/文件夹的 AI 分析
- 隔离区：已清理文件的恢复管理
- 历史记录：操作日志时间线
- 设置：LLM 配置、清理偏好、隔离区策略

### 3.2 CLI（命令行）

面向极客/开发者，快速高效、可脚本化。

**核心命令：**
```
bakubaku scan [path]        # 扫描指定目录或全盘
bakubaku clean              # 执行清理（交互式确认）
bakubaku clean --auto       # 仅清理安全级别最高的文件
bakubaku status             # 磁盘空间概览
bakubaku log                # 查看操作日志
bakubaku quarantine list    # 查看隔离区
bakubaku quarantine restore # 从隔离区恢复
bakubaku duplicates         # 重复文件检测
```

### 3.3 共享核心

CLI 和 GUI 共享同一个 `bakubaku-core` 引擎，记忆系统也共享 — 用 CLI 清理过的经验，GUI 里也能用上。

## 4. 非功能需求

### 4.1 性能

- 扫描速度：秒级（见 2.1 性能目标）
- 内存占用：< 200MB（Bakubaku 自身，不含 LLM 本地模型）
- 安装包大小：< 30MB（不含本地模型）

### 4.2 隐私与安全

- 所有文件分析在本地完成
- 发送给 LLM API 的数据仅限脱敏后的文件元数据
- 不做任何遥测上报
- 支持完全离线模式（使用本地模型）
- 用户可审查每批发送给 LLM 的数据

### 4.3 平台支持

- MVP 阶段：Windows（优先）
- 后续扩展：macOS、Linux

### 4.4 LLM 支持

- 云端 API：Claude（推荐）、OpenAI、其他兼容 API
- 本地模型：通过 Ollama 支持（Qwen、Llama 等）
- 用户可在设置中自由切换
